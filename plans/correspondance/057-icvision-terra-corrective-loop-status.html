Execution Update

The same Grace-reviewed evaluation set was confirmed for the comparison. The existing Autoclean/ICLabel baseline agreed with the expert-reviewed labels on 448 of 679 components, or 65.98%. The updated Terra prompt run agreed on 335 of 679 components, or 49.34%. The dominant Terra failure was not random degradation across every class; it was a systematic overcall of channel_noise: Terra predicted channel_noise 176 times while the expert-reviewed set contained 29 true channel_noise labels. That makes the next scientific move a targeted corrective loop, not broad prompt churn.

Corrective-loop decision

The first locked development subset was built to test the observed failure instead of inventing new ones: 132 unique components spanning the failure buckets that matter most, with overlapping bucket memberships preserved for analysis. The first stricter channel-noise prompt reduced some false positives and improved the failure-heavy subset overall, but it hurt true channel-noise detection and controls enough that it was rejected. The next active hypothesis is a smaller correction: treat channel noise as a class that requires electrode-localized evidence, not merely an odd topography or noisy image.

Guardrail: one observed failure, one exact change, one locked test, then keep, revert, or quarantine. This prevents the team from inventing problems and solving them theatrically.

Failure-mode interpretation

  • What seems real: Terra is over-weighting some visual signs as channel noise.
  • What is not proven: that Terra is globally incapable, that the image renderer is wrong, or that changing many prompt sections at once would improve accuracy.
  • What matters next: a corrective loop should improve channel-noise precision without sacrificing true channel-noise recall, eye/heart/muscle recognition, or ordinary brain controls.

Scope Limits

This record is a public summary. It does not include row-level labels, private image files, component-level examples, credential details, server locations, or publication-grade statistical claims. The numbers are development evidence for prioritizing the next loop, not manuscript evidence.

Conflict Status

No unresolved conflict is recorded here. The operating decision is to keep accuracy as the top objective while applying Ponytail minimalism: make the smallest testable classifier change tied to the dominant observed error.

Execution Decision

Do not optimize broadly yet. Continue with a bounded corrective loop for the channel-noise overcall, compare against the locked subset and the full Grace-reviewed set, and only promote a prompt change if it improves the target failure without degrading the baseline classes that already work.
