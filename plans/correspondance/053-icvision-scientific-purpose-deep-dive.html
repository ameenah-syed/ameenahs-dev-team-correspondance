Executive Answer

ICVision is trying to preserve meaningful brain-related EEG while identifying and optionally subtracting structured contamination. It does not calculate ICA. It accepts a previously computed ICA decomposition, renders evidence about each independent component, asks a vision-language model for a broad source label, and can use that result to recommend or perform component rejection during EEG reconstruction.

The pipeline is therefore an automated ICA-component review and artifact-rejection system, not a source-separation algorithm, anatomical localization method, diagnostic system, or proof that a component is neural.

1. The Scientific Problem: Scalp EEG Is a Mixture

Each scalp electrode records a mixture of electrical processes rather than one isolated brain source. The mixture may contain neural population activity, eye blinks and saccades, facial or neck muscle activity, cardiac electrical activity, power-line interference, electrode movement, bad-channel noise, and instrumentation noise.

A simplified model is:

x(t) = A s(t) + e(t)

x(t): measured electrode voltages
s(t): hidden source processes
A:    how each source projects onto the electrodes
e(t): residual noise and model mismatch

Independent component analysis estimates an unmixing matrix W so that s_hat(t) = W x(t). It searches for component activations that are as statistically independent and non-Gaussian as possible under an approximately linear and instantaneous mixing model. These are useful modeling assumptions, not proof that the recovered components are truly independent biological generators.

2. What an Independent Component Represents

An ICA component is a pair:

  1. a time course describing when the component is active; and
  2. a scalp projection describing how that activation appears across electrodes.

Component order, sign, and absolute scale are arbitrary. A red/blue scalp map may be polarity-inverted without changing the underlying reconstruction. A component is not automatically one neuron, one cortical region, or one physiological mechanism. One process can be split across multiple components, and one component can retain a mixture of neural and artifactual activity.

The repository requires a fitted ICA object or an EEGLAB dataset containing ICA weights; it does not fit ICA inside the classification pipeline. See core.py and utils.py.

3. What ICVision Automates

Precomputed ICA + continuous EEG
        ↓
Render evidence for each component
        ↓
Vision-language model assigns label, confidence, and explanation
        ↓
Policy decides retain, review, or exclude
        ↓
Selected component contributions may be removed during reconstruction

The intended benefit is faster and more standardized review of large numbers of components. Human component review is labor-intensive and depends on experience. Automating candidate labels can support triage and reproducibility, but only validated decision rules should control irreversible scientific choices.

The repository’s taxonomy is brain, eye, muscle, heart, line_noise, channel_noise, and other_artifact. See config.py.

5. Important Code-to-Scientific-Claim Mismatches

Line-noise evidence is hidden by the default PSD range

The prompt says that line_noise requires a sharp 50/60-Hz peak, but the default PSD ends at 45 Hz. Under the default rendering, the decisive evidence requested by the prompt is not visible.

The “ERP image” is not a genuine ERP image

The code mean-centers continuous activation, divides it into consecutive 1.5-second windows, and smooths across neighboring rows. The prompt calls those rows “trials,” but they are not experimental trials or event-aligned epochs. The display can show recurrent texture, but it cannot prove event-related or trial-to-trial consistency.

A brief time-series excerpt can be overinterpreted

The model sees only the first 2.5 seconds in the time-series panel. That may miss intermittent artifacts, while a coincidental two- or three-cycle pattern may look cardiac without demonstrating stable periodicity.

6. What the Vision-Language Model Actually Does

The model receives a written rubric and a rendered image. It performs prompted visual classification. It does not inspect the original multichannel voltage matrix, calculate ICA, correlate against dedicated EOG/ECG channels, perform dipole fitting, read event markers, or evaluate electrode impedance and acquisition context.

The returned confidence is a model-generated number, not a demonstrated calibrated probability. A reported value of 0.90 does not mean that comparable components have been shown to be correct 90% of the time. The core path places the selected class confidence into a score array rather than producing a validated seven-class probability distribution.

7. How Rejection Changes the EEG

When a component is rejected, its estimated activation is set to zero during back-projection:

s'_j(t) = 0 for rejected components
x_clean(t) = A s'(t)

This does not delete contaminated time points. It removes the rejected component’s estimated contribution at every time point. That can preserve usable data during a blink, but it also removes any meaningful neural activity contained in a mixed or incorrectly rejected component. The implementation applies the ICA exclusion list to a copy of the raw recording; see core.py.

8. Highest-Risk Current Behavior

Technical failure can masquerade as a confident artifact.

Several invalid-image, parsing, connection, authentication, or response failures can fall back to other_artifact with confidence 1.0. Because non-brain classes are eligible for automatic exclusion, a system failure can become a signal-removal decision. Failure must instead produce an explicit error status, confidence zero, and exclude_vision=false.

other_artifact is also scientifically heterogeneous: it may represent mixed components, unusual neural sources, poor decompositions, ambiguous evidence, or unrecognized artifacts. It is better suited to an abstention or review queue than automatic removal without supporting validation.

Relevant classification and exclusion behavior is implemented in api.py and core.py.

9. Appropriate Validation

A defensible validation should use frozen standardized evidence, multiple blinded EEG experts, adjudication for disagreements and mixed components, and complete held-out participants or recordings rather than randomly splitting components from the same recordings.

Evaluation should report per-class confusion matrices, sensitivity, precision, macro-F1, parse failures, abstentions, expert agreement, and confidence calibration. ICLabel can serve as a useful comparator, but neither ICLabel nor a single expert should be treated as unquestionable ground truth.

Most importantly, classification agreement and cleaning safety must be evaluated separately. The study must determine whether rejection attenuates the intended artifact while preserving the ERP, spectral, connectivity, or other neural measurements the study actually cares about.
